# 專案搜尋

在規劃或實作非小型（non-trivial）工作之前使用此流程。

## 步驟

1. 若存在，先閱讀 `ai/context/project-map.md`、`ai/context/architecture-map.md` 與 `ai/context/code-search-guide.md`。
2. 確定精確的搜尋詞：符號（symbol）、路由（route）、元件名稱、API 路徑、錯誤訊息。
3. 優先使用 `rg` 及具符號感知能力的工具，例如可用時的 Serena MCP。
4. 只閱讀最小範圍的相關檔案集合。
5. 依照 `ai/process/context-protocol.md` 產出情境包。
6. 僅在發現具持久性價值時，才更新 `ai/context/code-search-guide.md`。
