# UI Mockup 關卡

當任務涉及變更畫面、元件、互動流程或視覺狀態時，皆須使用此流程。

## 步驟

1. 建立或更新 `ai/templates/screen-spec.md`。
2. 列出所有必要狀態：預設、載入中、空狀態、錯誤、停用、權限不足、行動裝置版。
3. 產出 2-3 個 mockup 變體。
4. 依清晰度、資訊密度、實作複雜度與風險比較各變體。
5. 將決策記錄於 `ai/templates/mockup-decision.md`。
6. 在人工選擇變體之前，停止進行實作。

## 輸出

- 畫面規格書。
- 變體比較表。
- 建議採用的變體。
- 待釐清問題。
- 人工核准請求。
