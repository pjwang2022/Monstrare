# AI-Ready 任務卡範例

## Metadata

- 任務：為專案列表新增空狀態
- 上層規格：專案儀表板
- 狀態：就緒
- 風險等級：低
- Agent owner：實作 agent
- 人工核准者：產品負責人

## 目標

當使用者沒有任何專案時，顯示一個空狀態畫面，並附上清楚的「建立專案」動作。

## 情境包（Context Pack）

- 相關檔案：
  - `src/pages/projects/*`
  - `src/components/project-list/*`
- 既有模式：
  - 重複使用既有的按鈕與 modal 元件。
- 假設：
  - API 在沒有專案時已經會回傳空陣列。
- 未知事項：
  - 無。
- 允許變更的檔案：
  - 專案列表元件、相關測試檔。
- 不得觸碰：
  - 身分驗證、路由、資料庫 schema。

## 需求

- 顯示空狀態標題、簡短說明，以及建立動作。
- 點擊建立會開啟既有的建立專案 modal。
- 手機版排版不可溢出。

## 驗收標準

- 只有在專案數量為零時才出現空狀態。
- 既有的非空列表行為不受影響。
- 建立動作會開啟 modal。
- 測試涵蓋空與非空兩種情境。

## 驗證契約

- 單元測試：專案列表的空／非空狀態。
- 螢幕截圖：桌面版與手機版的空狀態。
- Lint：專案的 lint 指令。
- Build：專案的建置指令。
