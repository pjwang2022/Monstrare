# 情境協定（Context Protocol）

情境（context）是一種預算。要有意識地花用。

## 搜尋順序

1. 讀 `ai/context/project-map.md`。
2. 讀 `ai/context/architecture-map.md`。
3. 讀 `ai/context/code-search-guide.md`。
4. 搜尋精確的符號、路由、元件名稱、endpoint 路徑與錯誤文字。
5. 讀取最小範圍的相關檔案集合。
6. 有可用時使用語意工具，例如 Serena MCP。
7. 只有在證據不足時才擴大搜尋範圍。

## 情境包（Context Pack）

每個實作任務都應該以下列格式開始：

```text
任務：
目標：
相關檔案：
既有模式：
假設：
未知事項：
允許變更的檔案：
不得觸碰：
驗證指令：
風險等級：
```

## 搜尋規則

- 文字搜尋優先用 `rg`。
- 有可用時優先用具符號感知能力的工具。
- 除非任務範圍是整個 repo，否則不要讀整個 repository。
- 摘要或符號搜尋就夠用時，不要把大檔案整份貼進情境。
- 摘要發現並引用檔案路徑。
- 發現對專案有用的知識時，更新 `ai/context/code-search-guide.md`。

## 停止條件

遇到以下情況時要停下來請示方向：

- 搜尋發現多種可能的實作方式、各有不同取捨。
- 需求跟既有架構衝突。
- 缺少必要檔案。
- 任務涉及密鑰、身分驗證、金流、遷移或基礎設施。
- 預估範圍超出已核准的任務卡。
