# 完成定義（Definition of Done）

任務只有在「實作」與「驗證證據」兩者都存在時，才算完成。

## 必要證據

- 變更的檔案。
- 行為變更摘要。
- 執行過的指令。
- 測試輸出摘要。
- Lint／型別檢查（typecheck）／建置輸出（如適用）。
- UI 變更的螢幕截圖或視覺比對。
- 高風險變更的安全性備註。
- 已知限制。
- 後續任務。

## 不算完成

以下情況不得標記為完成：

- 測試被跳過卻沒有說明原因。
- 驗證只是「看起來沒問題」。
- UI 變更了卻沒有做螢幕截圖審查。
- 驗收標準沒有被檢查過。
- 實作變更了不相關的檔案。
- 發現的風險沒有負責人或後續處理。
