# 模型與 Agent 路由

這份文件定義的是決策權責。實際可用的模型取決於使用者的工具與訂閱方案。

## 高推理模型

用於：

- 模糊的產品決策。
- 系統設計。
- 跨服務變更。
- 架構取捨。
- 安全性敏感的推理。

範例：

- Claude Opus 級模型。
- 高推理等級的 Codex 模型。

## 快速通用模型

用於：

- 小型實作任務。
- 文件更新。
- 新增測試。
- 簡單重構。
- Mockup 草稿。

範例：

- Claude Sonnet 級模型。
- 快速版 Codex 模型。

## 審查 Agent

用於：

- PR 審查。
- Diff 審查。
- 安全性審查。
- 回歸分析。
- 測試缺口偵測。

## 路由規則

- 高風險任務不得在沒有審查的情況下交給低成本的實作模型。
- 不得讓同一個 agent 既負責實作、又是唯一的審查者。
- 雜訊多的搜尋或大範圍調查要用獨立的情境（context）進行。
- 規格書、情境包與驗收標準都清楚之後，優先使用成本較低的模型。
