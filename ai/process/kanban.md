# AI 看板

這個看板追蹤的是「這項任務對 AI agent 來說是否可以安全執行」，不只是單純的任務狀態。

這個看板的實際可用實作在 `tools/kanban/`（見 `tools/kanban/README.md`）。它是選用的工具，不是這份政策的強制要求——團隊可以用任何工具追蹤同樣的欄位。

## 欄位

```text
收件匣（Inbox）
-> 待釐清
-> 待產品核准
-> 待 UI Mockup
-> 待架構規劃
-> 待任務卡
-> AI 就緒
-> Agent 執行中
-> 待驗證
-> 待審查
-> 待人工驗收
-> 完成
```

## 欄位規則

### 收件匣（Inbox）

原始想法。尚未實作。

### 待釐清

問題、使用者、非目標或驗收標準還不清楚。

離場條件：

- 已有功能規格書草稿。
- 阻擋性問題已回答，或假設已記錄。

### 待產品核准

規格書可讀，但尚未核准。

離場條件：

- 人工已核准目標、非目標與驗收標準。

### 待 UI Mockup

涉及畫面或互動變更時需要。

離場條件：

- 已有畫面規格書。
- 已有 2-3 個 mockup 變體。
- 人工已選定一個方向。

### 待架構規劃

中高風險工作需要。

離場條件：

- 已列出可能變更的檔案。
- 已描述資料／API 契約。
- 安全性與測試風險已知。

### 待任務卡

計畫的顆粒度還不足以讓 agent 執行。

離場條件：

- 任務卡符合就緒定義。
- 每張卡都有驗證契約。

### AI 就緒

任務可以委派給 agent。

離場條件：

- 已指定 agent owner。
- 允許範圍已明確。

### Agent 執行中

Agent 正在實作一張已核准的任務卡。

離場條件：

- 已有 diff。
- Agent 已回報變更的檔案並嘗試驗證。

### 待驗證

實作已存在，但證據尚不完整。

離場條件：

- 已有驗證報告。
- 必要檢查已通過，或失敗已被記錄。

### 待審查

證據已存在，但審查關卡尚未完成。

離場條件：

- 必要審查者已核准或要求修改。

### 待人工驗收

任務已準備好交由人工做最終決定。

離場條件：

- 人工已接受、拒絕，或退回前一個欄位。

### 完成

已驗收且已驗證。後續事項另立新卡。

## WIP 上限

建議值：

- Agent 執行中：每位人工審查者 1-3 張。
- 待審查：最多 5 張。
- 待人工驗收：最多 5 張。

若審查欄位滿了，就停止建立新的 AI 工作。
