# 就緒定義（Definition of Ready）

任務只有在所有必要欄位都齊備時，才算「AI-ready」。

## 必要條件

- 目標明確具體。
- 使用者或系統行為清楚。
- 範圍有界。
- 已列出相關檔案或搜尋入口。
- 驗收標準可測試。
- 非目標明確列出。
- 已指定風險等級。
- 已定義驗證方法。
- 已記錄人工核准狀態。
- 若有前置任務（`dependsOn`），前置任務須全部為 `done` 才能把這張卡標記為就緒；有任何一個還沒完成，這張卡只能停在 `backlog` 或 `blocked`。

## UI 工作額外要求

- 畫面名稱。
- 目標視窗（viewport）假設。
- 狀態：預設、載入中、空狀態、錯誤、停用、權限不足、行動裝置版。
- 已讀過 `ai/context/design-system.md`，並列出本任務會重用的既有 token 與元件。
- 若需要的元件不在元件庫 inventory 裡，已標記為「需照既有風格補做並登記回 inventory」。
- 已產出 mockup 變體（皆以既有 token／元件組成）。
- 人工已選定一個變體。
- 已列出視覺驗收標準。
- 需要螢幕截圖驗證。

## 後端或資料工作額外要求

- API 契約。
- 資料模型變更。
- 驗證規則。
- 錯誤處理。
- 身分驗證與權限預期。
- 相關時的遷移與回滾備註。

## 高風險工作額外要求

高風險工作包含：身分驗證、權限、金流、密鑰、檔案處理、網路存取、
資料庫遷移、基礎設施，以及跨服務變更。

需要：

- 架構審查。
- 安全性審查。
- 測試策略審查。
- 人工核准。
