# 安全性檢查清單

無論任務風險等級高低，在撰寫程式碼的當下就要套用這份清單——不是只有高風險工作的正式安全性關卡（Security Gate）審查時才用。

- 對照 OWASP Top 10（權限控管失效、加密機制失效、注入攻擊、不安全的設計、安全設定錯誤、使用有漏洞或過時的元件、身分驗證失效、軟體與資料完整性失效、記錄與監控不足、SSRF）逐項檢查。
- 絕不以明文儲存密碼或密鑰。密碼要雜湊處理（例如 bcrypt/argon2），金鑰／權杖要從密鑰管理服務或環境變數載入，不要寫在原始碼裡。
- 驗證所有外部輸入。
- 檢查身分驗證邊界。
- 檢查每個受保護動作的授權。
- 不要記錄密鑰或敏感個人資料。
- 不要寫死（hardcode）憑證。
- 檢查檔案上傳、檔案路徑與指令執行行為。
- 檢查網路呼叫與外部 URL。
- 檢查注入風險：SQL、XSS、指令、樣板、路徑穿越（path traversal）。
- 視情況檢查 CSRF 與 CORS。
- 檢查依賴套件與供應鏈風險。
- 檢查遷移（migration）與回滾（rollback）安全性。
